Introduction

CSS has grown from a simple styling language into a powerful, declarative system for building entire interfaces. Modern CSS can produce responsive layouts, interactive components, smooth animations, design systems, accessible experiences and even some logic — all without a single line of JavaScript. This guide takes you from the fundamentals of pseudo-selectors through to production-grade architecture.

Each concept is presented in a consistent lesson card with the same structure: a definition, the official syntax, accepted values, a plain explanation, a live demonstration you can interact with, the exact HTML and CSS that produced it, browser-support notes, best practices, common mistakes and real-world use cases.

Why Advanced CSS Matters

⚡

Performance

A selector or property written well can shave milliseconds off every paint. Efficient CSS keeps animations at 60 fps and pages fast on mobile.

🎨

Design systems

Custom properties, naming conventions and component patterns let you build scalable, themeable design systems that grow with your product.

♿

Accessibility

Focus styles, reduced-motion support, color contrast and keyboard navigation are first-class concerns — not afterthoughts.

📱

Responsiveness

Media queries, fluid functions like clamp() and modern layout modules make interfaces adapt to any screen gracefully.

🔧

Maintainability

BEM, utility classes and CSS variables eliminate the dreaded specificity wars and make large stylesheets a joy to maintain.

🚀

Less JavaScript

Tabs, accordions, tooltips, dropdowns and themes are achievable in pure CSS — fewer dependencies, smaller bundles, faster sites.

SECTION 01

Pseudo-classes

A pseudo-class is a keyword added to a selector that specifies a special state of the selected element — for example :hover when a pointer is over it, or :nth-child() based on its position. They let you style dynamic states and document structure without extra markup.

:hover

Stable Interactive

Definition

Matches an element while the user pointers over it with a pointing device (mouse, trackpad, pen).

Syntax & Values

selector:hover { /* styles */ }

Live demonstration

Hover & click me Move the pointer over the button — then press & hold to see :active.

HTML

<a class="demo-hover" href="#">Hover & click me</a>

CSS

.demo-hover {
  background: var(--color-primary);
  transition: all 220ms ease;
}
.demo-hover:hover {
  background: var(--color-primary-700);
  transform: translateY(-3px) scale(1.04);
}
.demo-hover:active {
  transform: translateY(0) scale(0.97);
}

Browser support

Chrome ✅ Firefox ✅ Safari ✅ Edge ✅
✅

Best practice

Pair :hover with a transition so state changes feel smooth, not jarring.
⚠️

Common mistake

Hiding content only on :hover — touch devices have no hover. Always provide a tap fallback.

Real-world use cases

Buttons, navigation links, card lifts, dropdown menus, tooltips, image zoom effects, table-row highlighting, and anywhere a user can pause a pointer.

:active

Stable

Definition

Matches an element while it is being activated — typically while the mouse button is held down, or a key like Enter is pressed on a focused control.

Syntax

selector:active { /* styles while pressed */ }

Live demonstration

Hold the mouse button down (or press Space/Enter while focused) to see the active state.

HTML

<button>Press and hold</button>

CSS

button:active {
  transform: scale(0.95);
  background: var(--color-primary-700);
}

Browser support

All browsers ✅
✅

Best practice

Keep active feedback short (100–200ms) so it feels like a tactile press.
⚠️

Common mistake

Forgetting the LVHA order: :link :visited :hover :active. Out of order, one state overrides another.

Use cases

Tactile button presses, drag handles, press feedback, and indicating that an action has been received.

:focus · :focus-visible · :focus-within

StableAccessibility

Definitions

:focus
Matches while the element has focus — keyboard or mouse.
:focus-visible
Matches only when the browser heuristically decides focus should be visible (usually keyboard). Avoids the ugly outline on mouse clicks.
:focus-within
Matches a parent when any descendant is focused — perfect for highlighting a whole form field group.

Syntax

input:focus-visible {
  outline: none;
  box-shadow: 0 0 0 4px var(--color-primary-soft);
  border-color: var(--color-primary);
}
.field:focus-within { background: var(--color-primary-soft); }

Live demonstration

Click with the mouse — no ring. Press Tab from the keyboard — the ring appears because :focus-visible only shows for keyboard users.

HTML

<input class="demo-focus" type="text" />

CSS

.demo-focus:focus-visible {
  outline: none;
  border-color: var(--color-primary);
  box-shadow: 0 0 0 4px var(--color-primary-soft);
}

Browser support

:focus — all:focus-visible — 2019+:focus-within — 2018+
✅

Best practice

Prefer :focus-visible over :focus. Never use outline: none without providing a replacement ring.

⚠️

Common mistake

Removing the outline globally — this fails WCAG 2.4.7 (Focus Visible).

Use cases

Keyboard accessibility, form-field highlighting, focus traps in modals, and styling interactive elements without annoying mouse users.

:link · :visited

Stable

Definitions

:link
Matches unvisited hyperlinks that have an href.
:visited
Matches hyperlinks the user has already visited. Browsers limit which properties you can style (privacy).

Syntax

a:link    { color: var(--color-primary); }
a:visited { color: var(--color-primary-700); }

Live demonstration

An unvisited-style link Likely-visited link

HTML

CSS

a:visited { color: #3730a3; }

Browser support

All browsers ✅
✅

Best practice

Keep visited links distinguishable but not jarring; respect user privacy limits.

⚠️

Common mistake

Trying to set background or font-size on :visited — browsers ignore most properties for privacy.

Use cases

Search results, navigation, history-aware reading flows.

:checked · :disabled · :enabled

Stable

Definitions

:checked
Matches a radio, checkbox, or option that is currently selected.
:disabled
Matches form controls marked disabled — not editable or submittable.
:enabled
The inverse — a usable form control.

Live demonstration

Forgetting that disabled inputs are still submitted if they have a name & value — set readonly instead if data must persist.

Custom checkboxes/radios, switch toggles, theme switches, disabled form states, conditional UI.

Accepted values

:target · :root

Stable

Definitions

:target
Matches the element whose id equals the current URL fragment (#…).
:root
Matches the document root — <html>. The conventional place to declare global CSS variables.

:not() · :is() · :where()

StableModern

Definitions

:not()
Matches anything except the selectors inside.
:is()
Matches any of the selectors inside; takes on the highest specificity of its arguments.
:where()
Identical to :is() but with zero specificity — perfect for safe defaults.

Live demonstration

Internal link External link Internal link

CSS

/* :not() — style everything except .external */
a:not(.external) { color: var(--color-primary); }

/* :is() — one rule for many headings */
:is(h1, h2, h3) { color: var(--color-heading); }

/* :where() — zero-specificity base style */
:where(button) { padding: .5rem 1rem; }

Browser support

:not() all:is()/:where() 2021+
✅

Best practice

Use :where() for resets & base styles so a single class can override them without specificity wars.

⚠️

Common mistake

Using :is() when you wanted a low-specificity default — it adopts the highest specificity of its arguments.

Use cases

DRY selector lists, exception handling ("everything except…"), shareable base styles, modern resets.

:has() — the "parent selector"

Baseline 2023Modern

Definition

:has() matches an element that contains a descendant matching the argument — historically impossible in CSS until 2023. It's the long-awaited "parent selector".

Live demonstration

This card has only text. This card has an image — it gets a primary tint!

CSS

/* Style a card that contains an <img> */
card:has(img) {
  background: var(--color-primary-soft);
  border-color: var(--color-primary);
}

/* Hide section if it has no articles */
section:not(:has(article)) { display: none; }

Browser support

Chrome 105+Safari 15.4+Firefox 121+
💡

Heads-up

Still not in older browsers (≈2022 and earlier). Use as a progressive enhancement, not for critical layout.

Use cases

Conditional styling based on content, "card with image" layouts, form validation states (:has(input:invalid)), and reducing JavaScript.

SECTION 02

Pseudo-elements

Unlike pseudo-classes (which select an existing element in a state), pseudo-elements create or style a part of an element — such as generated content, the first letter, or the selection highlight. They use a double-colon syntax: ::name.

::before · ::after

Stable

Definition

Insert a generated, anonymous child element at the start (::before) or end (::after) of an element's content. They only appear if the content property is set (even empty "").

Syntax

.badge::before {
  content: "★ ";
  color: gold;
}

Live demonstration

Featured

HTML

<span class="demo-pe">Featured</span>

CSS

.demo-pe::before { content: "★"; }
.demo-pe::after  {
  content: "";
  border: 2px dashed var(--color-primary);
}

Browser support

All browsers ✅
✅

Best practice

Use for decorative content only — never for content that screen-reader users need (it can be inconsistent).

⚠️

Common mistake

Forgetting content: ""; — without it, the pseudo-element won't render at all.

Use cases

Icons, badges, clear-fix, tooltips, decorative borders, counters, clearfix hacks.

::first-letter · ::first-line

Stable

Live demonstration

Long ago, in a stylesheet far away, a typographer decided that the first letter of a paragraph deserved to stand out. This technique — the drop cap — still adds elegance to articles today.

This is the first line of a paragraph and it is bold. This second line returns to normal weight. Notice how the styling adapts automatically even as the viewport reflows.

CSS

  color: var(--color-primary);

.demo-firstline::first-line {
  font-weight: 700;

Browser support

All browsers ✅

Use cases

Editorial drop caps, magazine-style articles, lead-paragraph styling.

::selection

Stable

Styles the portion of text the user has highlighted/selected. Only a handful of properties are allowed: color, background, text-shadow, and a few others.

SECTION 03

CSS Variables (Custom Properties)

Custom properties let you store values in named tokens and reuse them. They cascade, are inherited, can change at runtime (great for themes) and accept any valid CSS value — including other variables.

Declaring & using custom properties

Stable

Syntax

/* Declare on :root for global scope */
:root {
  --color-primary: #4f46e5;
  --space-4: 1rem;
}

/* Use with var() anywhere */
.btn {
  background: var(--color-primary);
  padding: var(--space-4);
}

/* Fallback value if --x is undefined */
color: var(--text, #333);

Variable scope & theming demo

Default Warm token Cool token Danger token

Each button reuses the same rule — only the tokens change locally.

CSS

.demo-theme {
  --btn-bg: var(--color-primary);
  --btn-fg: #fff;
  background: var(--btn-bg);
  color: var(--btn-fg);
}
.demo-theme.warm   { --btn-bg: orange; }
.demo-theme.danger { --btn-bg: red; }
✅

Best practice

Name tokens semantically (--space-gutter, not --blue) so themes can swap values cleanly.

⚠️

Common mistake

Always providing a fallback bloats the codebase — only fall back when the variable might be undefined.

Use cases

Design systems, dark/light themes, component variants, responsive spacing (--space-4 resized in media queries), live theming.

Mini design system

Reusable tokens

This very page is a working design system. The :root in index.css declares every color, spacing, radius, shadow and motion value as a token. Every component reads from those tokens — that's why theming (including dark mode) is a one-place change.

:root {
  --color-primary: #4f46e5;
  --space-4: 1rem;
  --radius-lg: 16px;
  --shadow: 0 6px 24px rgba(15,27,45,.08);
}
@@media (prefers-color-scheme: dark) {
  :root { --color-primary: #818cf8; }
}
SECTION 04

CSS Functions

Functions turn CSS from static into a computational language. calc() does math; min()/max()/clamp() pick responsive values; var() reads tokens; url() loads assets; attr() reads HTML attributes.

calc()

Definition

Performs math on mixed units — essential when combining % with px, rem, vh, etc.

Live demonstration

width: calc(100% − 40px)

CSS

.demo-calc {
  width: calc(100% - 40px);   /* note spaces around - */
  padding: calc(var(--space-4) * 2);
}
⚠️

Common mistake

Forgetting the spaces around + and - — calc(100%-40px) is parsed as an invalid value.

clamp() · min() · max()

Definitions

min(a, b, …)
Picks the smallest — caps a maximum size.
max(a, b, …)
Picks the largest — sets a minimum size.
clamp(min, ideal, max)
Fluid value that never goes below min or above max.

Live demonstration

Fluid heading — resize the window!
Container is at most 480px wide

CSS

.demo-clamp  { font-size: clamp(1.25rem, 5vw, 3rem); }
.demo-minmax { width: min(100%, 480px); }
✅

Best practice

clamp() is the modern way to do fluid typography — no media queries needed.

var() · url() · attr()

Definitions

var(--x, fallback)
Reads a custom property; fallback is used if undefined.
url("…")
Loads an external resource (image, font, cursor).
attr(name)
Reads an HTML attribute value. Limited browser support for anything other than content.

Live demonstration (attr-driven tooltip)

Hover this link

CSS

[data-tip]:hover::after {
  content: attr(data-tip);
  background: #0f172a;
}
.logo { background-image: url("logo.svg"); }
SECTION 05

Media Queries

Media queries let styles adapt to the device — viewport width, orientation, resolution, color scheme, motion preference, input type and more. They are the backbone of responsive design.

@media & common breakpoints

Mobile-first vs desktop-first

ApproachPatternWhen to use
Mobile-firstmin-width queries add styles as screens growModern default — smaller, faster CSS
Desktop-firstmax-width queries remove styles as screens shrinkLegacy codebases

Common breakpoints

/* Mobile-first: base = mobile styles */
@@media (min-width: 640px)  { /* small tablets */ }
@@media (min-width: 768px)  { /* tablets      */ }
@@media (min-width: 1024px) { /* laptops      */ }
@@media (min-width: 1280px) { /* desktops     */ }

Live demonstration

1
Resize the window to watch the grid reflow
2
3
4

Orientation · Resolution · Hover · Pointer

Examples

/* Landscape phones */
@@media (orientation: landscape) and (max-height: 500px) { }

/* High-DPI / retina screens */
@@media (min-resolution: 2dppx) { }

/* Devices WITHOUT hover (touch) */
@@media (hover: none) { --hover-style: none; }

/* Coarse pointer (finger, not mouse) */
@@media (pointer: coarse) { --tap-size: 48px; }
✅

Best practice

Use (hover: hover) and (pointer: fine) to gate desktop-only interactions, rather than sniffing screen width.

Dark mode · Print · Reduced motion

/* Honor OS dark-mode preference */
@@media (prefers-color-scheme: dark) {
  :root { --color-bg: #0a1020; }
}

/* Print: hide chrome, force colors */
@@media print {
  .header, .sidebar, .footer { display: none; }
  body { background: #fff; }
}

/* Respect reduced motion */
@@media (prefers-reduced-motion: reduce) {
  * { animation: none !important; transition: none !important; }
}
💡

Try it

This whole site already responds to dark mode and print — change your OS theme or print-preview this page.

SECTION 08

CSS Architecture

As stylesheets grow, structure matters. These patterns keep large codebases predictable, themeable and maintainable.

Reset vs Normalize

/* Minimal modern reset */
*, *::before, *::after { box-sizing: border-box; }
* { margin: 0; }
body { line-height: 1.6; }
img { max-width: 100%; }

A reset zeroes out all default styles. Normalize preserves sensible defaults and corrects browser inconsistencies. Pick one — this page uses a tiny reset.

BEM naming

.card           /* Block */
.card__title    /* Element  (double underscore) */
.card--featured /* Modifier (double dash)     */

BEM makes relationships explicit and keeps specificity flat — no nesting wars.

Utility classes

.u-mt-4  { margin-top: var(--space-4); }
.u-center { text-align: center; }
.u-muted  { color: var(--color-text-muted); }

Single-purpose helpers compose quickly and avoid repeating yourself. This site uses several (see Section 14 of the stylesheet).

Component-based CSS

Group all rules for a component together — header, body, footer — and let CSS variables drive variants. That is exactly the structure of index.css: tokens → reset → base → layout → components → utilities → responsive.

File organization

styles/
 ├─ 01-tokens.css     /* variables */
 ├─ 02-reset.css
 ├─ 03-base.css       /* typography */
 ├─ 04-layout.css
 ├─ 05-components.css
 └─ 06-utilities.css

Split files by responsibility and concatenate at build time. (This single-file site mimics the same order via comments.)

Scalable CSS strategies

  • Keep specificity flat — prefer classes over IDs/tags.
  • Avoid !important; refactor instead.
  • One responsibility per class.
  • Theme with variables, not overrides.
  • Name things by intent, not appearance.
SECTION 09

Accessibility

Accessible CSS isn't a feature — it's a baseline. These techniques ensure users of all abilities can perceive and operate your UI.

Visible focus styles

Never remove outline without a replacement. Use :focus-visible so keyboard users get a clear ring without bothering mouse users.

a:focus-visible {
  outline: 3px solid var(--color-primary);
  outline-offset: 2px;
}

Color contrast

Aim for WCAG AA: 4.5:1 for body text, 3:1 for large text and UI components. Never rely on colour alone to convey meaning.

✅

Tools

Use browser DevTools contrast checker or the WAVE extension.

Keyboard navigation

Every interactive element must be reachable via Tab and operable via Enter/Space. Maintain a logical tab order with sensible DOM order.

Reduced motion

@@media (prefers-reduced-motion: reduce) {
  * { animation: none !important; transition: none !important; }
}

Vestibular disorders can make motion painful — always respect this preference. This page does.

Readable typography

  • Body text ≥ 16px (1rem).
  • Line-height between 1.4 and 1.7.
  • Line length 45–75 characters.
  • Avoid fully justified text for screen readers.

ARIA-friendly styling

Style based on semantic attributes — [aria-expanded], [aria-current], [disabled] — instead of extra classes. This keeps behaviour and appearance in sync.

[aria-expanded="true"] .icon { transform: rotate(90deg); }
[aria-current="page"] { color: var(--color-primary); }
SECTION 10

Performance

CSS directly affects how fast a page paints and how smoothly it animates. A few habits keep things buttery.

Efficient selectors

Modern browsers read selectors right-to-left. Over-deep descendant selectors (.nav ul li a span) cost more than a single class (.nav-link).

Repaints vs reflows

Repaint
Pixel re-draw without layout change (color, background).
Reflow
Layout recalculated (width, height, position). Far more expensive.

Optimising animations

Animate only transform and opacity — they run on the GPU and skip layout/paint. Avoid animating top, left, width, box-shadow.

/* ✅ GPU-friendly */
.card:hover { transform: translateY(-4px); }

/* ❌ triggers reflow */
.card:hover { top: -4px; }

Reducing CSS size

  • Use CSS variables to avoid repeating values.
  • Delete unused selectors and dead code.
  • Minify for production.
  • Prefer shorthand properties.

contain & content-visibility

.card { contain: layout paint style; }
.feed-item { content-visibility: auto; }

Tell the browser an element is independent so it can skip work outside the viewport — huge wins on long lists.

Reusable classes

Composition beats duplication. Build small, single-purpose classes and compose them — fewer lines, fewer repaints, easier maintenance.

SECTION 11

Best Practices

The habits that separate hobbyist CSS from professional CSS.

📝 Comment generously

Mark sections, explain non-obvious hacks, and document token meanings. Future-you will thank you.

🏷️ Consistent naming

Pick a convention (BEM, utility-first, suits your team) and stick to it. This page mixes BEM blocks with u- utilities.

♻️ Build reusable components

If you've written it twice, extract it. Variants via modifiers or tokens, not copy-paste.

🧩 Maintainability

Flat specificity, no !important, variables for theming. Keep the cascade your friend.

📱 Responsive-first

Start mobile, enhance upward. Use clamp() and min()/max() to replace many media queries.

🌐 Cross-browser testing

Test on real devices. Use feature queries (@@supports) and provide graceful fallbacks.

🎨 Tokenise everything

Colors, spacing, radii, shadows and motion should all be variables. Brand changes become one edit.

⚖️ Use !important sparingly

It breaks the cascade. If you need it, your specificity strategy is wrong — refactor.

⭐

The golden rule

Write CSS for the next developer who will read it — not for the computer. Clarity beats cleverness.
